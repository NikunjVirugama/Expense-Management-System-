What is a Stack?

A Stack is a linear data structure that follows the LIFO (Last In, First Out) principle. Think of it like a stack of plates - you can only add or remove plates from the top!

LIFO Principle

Last In, First Out means the most recently added element is the first one to be removed. This makes stacks perfect for scenarios where you need to reverse the order of operations.

Key Operations

Master the four fundamental stack operations: Push (add), Pop (remove), Peek (view top), and Display (show all elements). Each operation is visualized interactively!

How Push Works

  1. Check if the stack is full (Stack Overflow condition)
  2. If not full, increment the top pointer
  3. Add the new element at the top position
  4. The new element becomes the top of the stack
void push(int x) { if(top == MAX-1) { cout << "Stack Overflow"; } else { stack[++top] = x; cout << "Pushed " << x; } }

Time Complexity: O(1)

Push operation takes constant time as we only need to add one element at the top.

How Pop Works

  1. Check if the stack is empty (Stack Underflow condition)
  2. If not empty, store the top element value
  3. Decrement the top pointer
  4. Return the stored value
int pop() { if(top == -1) { cout << "Stack Underflow"; return -1; } else { int x = stack[top--]; return x; } }

Time Complexity: O(1)

Pop operation takes constant time as we only need to remove the top element.

How Peek Works

  1. Check if the stack is empty
  2. If not empty, return the top element value
  3. Do NOT remove or modify the stack
  4. The stack remains unchanged after peek
int peek() { if(top == -1) { cout << "Stack is empty"; return -1; } else { return stack[top]; } }

Time Complexity: O(1)

Peek operation takes constant time as we only access the top element without any modifications.

How Display Works

  1. Check if the stack is empty
  2. If empty, display "Stack is empty"
  3. If not empty, traverse from top to bottom
  4. Print each element in LIFO order
void display() { if(top == -1) { cout << "Stack is empty"; } else { cout << "Stack elements: "; for(int i = top; i >= 0; i--) { cout << stack[i] << " "; } } }

Time Complexity: O(n)

Display operation takes linear time as we need to traverse all n elements in the stack.

Undo/Redo in Text Editors

Text Editor with Undo/Redo Stack

Undo Stack (Recent actions at top):

How it works: Every action you perform is pushed onto the undo stack. When you hit undo, the last action is popped from the undo stack and pushed onto the redo stack. This allows you to go back and forth through your editing history using the LIFO principle!

Call Stack in Recursion

Factorial Function Call Stack

Function Call Stack:

How it works: When a recursive function calls itself, each call is pushed onto the call stack. When a function returns, it's popped from the stack. This visualization shows how factorial(n) creates a stack of function calls that resolve in LIFO order!

int factorial(int n) { if(n <= 1) { return 1; } return n * factorial(n-1); }
